[Problem](https://leetcode.com/problems/contiguous-array/)


The idea is that we can turn 0 into -1 so that the problem is reduced to finding the longest subarray with sum 0. One observation is that if the `prefix_sum[i]` equals to `prefix_sum[j]`, the subarray starting from `i+1` and ending at `j` has a sum of 0. Therefore, we need to find the two indices that are furthest away from each other with the same prefix sum. To achieve this, we can store the first occurence of each prefix sum in a hash map. This solution is the same as the Editorial's approach 2.

How can we notice this type of problem? The cue is when we see a phrase like "equal number of X and Y", or when the problem requires something like "balance", "equilibrium", or "same count", we may need to use the prefix sum technique with presumably a hash map.

class Solution:
    def findMaxLength(self, nums: List[int]) -> int:
        prefix = 0
        first_idx = {0: -1}   # prefix sum -> earliest index
        max_len = 0

        for i, num in enumerate(nums):
            # Step 1: convert
            if num == 0:
                prefix -= 1
            else:
                prefix += 1

            # Step 2: check if seen before
            if prefix in first_idx:
                length = i - first_idx[prefix]
                max_len = max(max_len, length)
            else:
                first_idx[prefix] = i  # store first occurrence only

        return max_len
