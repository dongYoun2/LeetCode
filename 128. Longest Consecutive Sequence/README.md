## Hash Map Solution

This is a standard solution for this problem. it uses a Python `set` and find the start of the sequence by checking if the previous number is not in the set. Once the start of the sequence is found, it checks the length of the sequence by keep incrementing the current number by 1 until the next number is not in the set.

One good takeaway from this solution is that not all duplicated `for` or `while` loops have $O(n^2)$ time complexity. Because in this case, the inner `while` loop can only be executed at most $n$ times.

cf.) The below code is generated by ChatGPT.

[Submission](https://leetcode.com/problems/longest-consecutive-sequence/submissions/1775443954/)—Runtime: 43 ms (beats 84.81%), Memory: 33.3 MB (beats 91.85%)

- TC: $O(n)$, where $n$ is the number of elements in `nums`
- SC: $O(n)$ (`num_set`)

```python
class Solution:
    def longestConsecutive(self, nums: List[int]) -> int:
        if not nums:
            return 0

        num_set = set(nums)       # Unique numbers for O(1) lookup
        longest_streak = 0

        for num in num_set:
            # Start only if it's the beginning of a sequence
            if num - 1 not in num_set:
                current_num = num
                current_streak = 1

                while current_num + 1 in num_set:
                    current_num += 1
                    current_streak += 1

                longest_streak = max(longest_streak, current_streak)

        return longest_streak

```

## Union-Find Solution (Disjoint Set)


In the code below, we can notice that we attach the smaller tree (component) to the larger tree during the union operation so that the root of the bigger tree stays the root, and its size increases by the smaller tree's size. This is a key idea to keep the tree height small, and thus, we can perform the **find** operation faster (near constant time), which is crucial for the **amortized time complexity**.


cf.) The below code is generated by ChatGPT.


[Submission](https://leetcode.com/problems/longest-consecutive-sequence/submissions/1966189333/)—Runtime: 195 ms (beats 5.07%), Memory: 66.78 MB (beats 5.11%)

- TC: ~$O(n)$ (amortized, almost constant per operation)
- SC: $O(n)$ 

```python
class Solution:
    def longestConsecutive(self, nums: List[int]) -> int:
        if not nums:
            return 0

        nums = set(nums)                 # size ≤ n
        parent = {x: x for x in nums}    # each node is its own parent
        size = {x: 1 for x in nums}      # component sizes

        def find(x):
            # path compression
            if parent[x] != x:
                parent[x] = find(parent[x])
            return parent[x]

        def union(x, y):
            rx, ry = find(x), find(y)
            if rx == ry:
                return
            # union by size (attach smaller → bigger)
            if size[rx] < size[ry]:
                rx, ry = ry, rx
            parent[ry] = rx
            size[rx] += size[ry]

        # connect consecutive numbers
        for x in nums:
            if x + 1 in nums:
                union(x, x + 1)

        return max(size.values())

```
