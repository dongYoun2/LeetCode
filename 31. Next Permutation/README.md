## Solution

The algorithm is as follows:
1. Find where the decreasing order breaks from the right; find the pivot.
   - Why? Everything to the right is already the largest possible order.
2. Swap the pivot with the next larger element from the right.
3. Make the right part smallest; reverse the suffix to make it in an ascending order.

The edge case is when the entire array is in a decreasing order. In this case, we simply need to reverse the entire array, which is the smallest permutation.

The intuition is that we want the next slightly bigger arrangement, not a big jump. If we think of permutations in a dictionary order, we can often see cases where **the suffix is decreasing**, which means that the suffix is **already the largest possible order**. Therefore, we cannot increase the permutation by modifying the suffix. The only way to increase is to change something to the left (prefix).

This algorithm works, is natural, and guarantees the next permutation because we:
1. Change the rightmost possible position
2. Increase it by the smallest amount
3. Make the rest as small as possible

class Solution:
    def nextPermutation(self, nums: List[int]) -> None:
        """
        Do not return anything, modify nums in-place instead.
        """
        n = len(nums)

        # 1. Find pivot index i such that nums[i] < nums[i+1]
        i = n - 2
        while i >= 0 and nums[i] >= nums[i + 1]:
            i -= 1

        # 2. If pivot exists, swap with next larger element
        if i >= 0:
            j = n - 1
            while nums[j] <= nums[i]:
                j -= 1
            nums[i], nums[j] = nums[j], nums[i]

        # 3. Reverse the suffix starting at i+1
        left, right = i + 1, n - 1
        while left < right:
            nums[left], nums[right] = nums[right], nums[left]
            left += 1
            right -= 1
