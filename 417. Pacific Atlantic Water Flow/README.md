Below solutions are generated by ChatGPT. In both solutions, though the top-left corner cell and bottom-right corner cell are processed twice in the Pacific and Atlantic processing, respectively, `visited` set handles this, so the codes don't break.

The main idea is to perform BFS (or DFS) twice, one starting from the border cells of the Pacific and another from the border cells of the Atlantic. Then, simply return the cells that can reach both oceans.


## BFS solution

[Submission](https://leetcode.com/problems/pacific-atlantic-water-flow/submissions/1946647780/)—Runtime: 26 ms (Beats 76.07%), Memory: 20.64 MB (Beats 94.92%)

- TC: $O(m \cdot n)$, where $m$ is the number of rows and $n$ is the number of columns in the grid. (Each cell can be visited at most once per ocean.)
- SC: $O(m \cdot n)$

```python
from collections import deque

class Solution:
    def pacificAtlantic(self, heights):
        if not heights:
            return []

        m, n = len(heights), len(heights[0])

        def bfs(starts):
            visited = set()
            q = deque(starts)

            # initial queue and visited
            for r, c in starts:
                visited.add((r, c))

            while q:
                r, c = q.popleft()

                for dr, dc in [(1,0),(-1,0),(0,1),(0,-1)]:
                    nr, nc = r + dr, c + dc

                    if (
                        0 <= nr < m and
                        0 <= nc < n and
                        (nr, nc) not in visited and
                        heights[nr][nc] >= heights[r][c]
                    ):
                        visited.add((nr, nc))
                        q.append((nr, nc))

            return visited

        # Pacific borders (top row + left column)
        pacific_starts = [(0, c) for c in range(n)] + [(r, 0) for r in range(m)]

        # Atlantic borders (bottom row + right column)
        atlantic_starts = [(m-1, c) for c in range(n)] + [(r, n-1) for r in range(m)]

        pacific = bfs(pacific_starts)
        atlantic = bfs(atlantic_starts)

        return list(pacific & atlantic)

```



## DFS solution

[Submission](https://leetcode.com/problems/pacific-atlantic-water-flow/submissions/1946647881/)—Runtime: 25 ms (Beats 78.38%), Memory: 22.15 MB (Beats 53.34%)

- TC: $O(m \cdot n)$
- SC: $O(m \cdot n)$

```python
class Solution:
    def pacificAtlantic(self, heights):
        if not heights:
            return []

        m, n = len(heights), len(heights[0])

        pacific = set()
        atlantic = set()

        def dfs(r, c, visited):
            visited.add((r, c))

            for dr, dc in [(1,0),(-1,0),(0,1),(0,-1)]:
                nr, nc = r + dr, c + dc

                if (
                    0 <= nr < m and
                    0 <= nc < n and
                    (nr, nc) not in visited and
                    heights[nr][nc] >= heights[r][c]
                ):
                    dfs(nr, nc, visited)

        # Pacific borders
        for c in range(n):
            dfs(0, c, pacific)
        for r in range(m):
            dfs(r, 0, pacific)

        # Atlantic borders
        for c in range(n):
            dfs(m-1, c, atlantic)
        for r in range(m):
            dfs(r, n-1, atlantic)

        return list(pacific & atlantic)

```
